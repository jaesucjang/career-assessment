# 고등학교 1학년 전공 계열 선택 가이드 웹서비스 개발 계획

## 프로젝트 개요

- **목적**: 고1 학생이 모바일에서 간단한 설문에 응답하면, 적성에 맞는 전공 계열 1~3지망을 추천받는 웹페이지
- **로그인**: 불필요 (익명, 세션 기반)
- **디자인 컨셉**: 귀엽고 깜찍한 UI (파스텔톤, 둥근 카드, 이모지/일러스트 활용)
- **기술 스택**: HTML + CSS + Vanilla JS (단일 페이지, 서버 불필요, 정적 호스팅 가능)

## 페이지 흐름 (총 4단계)

### Step 1: 인트로 페이지
- 귀여운 일러스트 + 제목: "나의 전공 계열 찾기 🔮"
- 부제: "간단한 질문에 답하고, 나에게 맞는 전공을 찾아보세요!"
- [시작하기] 버튼

### Step 2: 설문 페이지 (21문항)
- **한 화면에 1문항씩** 표시 (모바일 최적화)
- 프로그레스바로 진행률 표시 (1/21 ~ 21/21)
- 5점 리커트 척도: 이모지 버튼으로 표현
  - 😑 전혀 아니다 (1점)
  - 😐 아니다 (2점)
  - 🙂 보통이다 (3점)
  - 😊 그렇다 (4점)
  - 🤩 매우 그렇다 (5점)
- 이전/다음 네비게이션
- 스와이프 제스처 지원 (선택사항)

### Step 3: 결과 페이지
- 계열별 점수 계산 후 상위 3개 추천
- **1지망 / 2지망 / 3지망** 카드 형태로 표시
  - 각 카드에: 계열명, 대표 이모지, 점수 바, 추천 이유 한 줄
- 하단에 "가이드 보기" 버튼 → 예시 가이드 팝업
- 결과 공유 버튼 (URL 복사 또는 캡처 안내)

### Step 4: 선택 확정 페이지
- 결과를 참고하여 **직접** 1지망/2지망/3지망 선택
- 드롭다운 또는 카드 클릭 방식으로 27개 계열 중 선택
- 추천 결과와 다른 선택도 가능 (자유 의지 존중)
- [제출하기] 버튼 → 최종 확인 팝업
- 제출 후 "완료!" 애니메이션

### 채점 방식
1. 각 문항 응답(1~5점)을 매핑된 계열에 가중 합산
2. 주 매핑 계열: 가중치 1.0 / 부 매핑 계열: 가중치 0.5
3. 계열별 총점을 내림차순 정렬
4. 상위 3개를 1~3지망으로 추천
5. 동점 시: 문항 번호가 작은 쪽 우선 (인문→사회→자연 순)

### 특수 규칙
- 모든 영역 점수가 고르면 → "자율전공계열" 추천 (예시1 반영)
- 예체능 관련 문항이 없으므로 → 결과 페이지에서 "예체능에 관심 있다면?" 별도 안내
- "기타"는 추천 결과에 포함하지 않음 (직접 선택 시에만 가능)

## 구현 순서

### Phase 1: 기본 구조 (index.html + style.css)

4. 21개 문항 데이터 구조 정의
5. 문항별 화면 렌더링 (1문항씩)

7. 프로그레스바 & 네비게이션

## 예시 가이드 팝업 내용

결과 페이지에서 "선택 팁 보기" 버튼 클릭 시 표시:

- 💡 진로를 아직 결정하지 못했다면 → **자율전공계열**
- 💡 사관학교를 지망한다면 → **국방계열**
- 💡 의학/치의학/한의학/약학/수의학 → **의약학계열**
- 💡 간호학/물리치료/언어치료 → **보건간호학계열**
- 💡 교대/사범대 → **교육학계열**
- 💡 정치외교 vs 미디어 고민 중 → **사회과학계열**로 넓게!
- 💡 수학과 지망 → **수리통계학계열** (구체적)
- 💡 컴퓨터 자체를 만드는 기술 → **정보컴퓨터공학계열**
- 💡 데이터로 AI를 만드는 기술 → **데이터인공지능공학계열**
- 💡 위 항목 모두 해당 없음 → **기타** (최후의 선택)

## 참고사항

- **서버 불필요**: 모든 로직이 클라이언트 JS에서 동작, GitHub Pages 등으로 바로 배포 가능
- **데이터 저장 없음**: 개인정보 수집 없이 브라우저 세션에서만 동작
- **예체능 문항 부재**: 설문에 예체능 관련 문항이 없으므로, 결과 페이지에서 별도 안내 문구 추가
- **기타 계열**: 추천 알고리즘에 포함하지 않고, 직접 선택 시에만 선택 가능
